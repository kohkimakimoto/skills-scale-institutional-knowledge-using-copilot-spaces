## Purpose
Provide a structured onboarding experience for new team members across all roles. This checklist ensures consistent knowledge transfer and accelerates time-to-productivity. Created to address gaps identified in [issue #4](https://github.com/kohkimakimoto/skills-scale-institutional-knowledge-using-copilot-spaces/issues/4).

## Before Day 1 (Hiring Manager / PM)
- [ ] Prepare workspace and access credentials
- [ ] Add to team communication channels (Slack, email lists)
- [ ] Share project documentation repository location
- [ ] Assign onboarding buddy/mentor
- [ ] Schedule first-week meetings

---

## Week 1: Foundation

### For All Roles
- [ ] Review [OctoAcme Roles and Personas](octoacme-roles-and-personas.md) to understand team structure
- [ ] Read [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Meet with team lead and onboarding buddy
- [ ] Set up development environment and tools (if applicable)
- [ ] Review current project status and roadmap
- [ ] Attend daily standups as observer
- [ ] Complete security and compliance training

### Role-Specific Week 1 Tasks

#### Developers
- [ ] Clone project repositories and verify build process
- [ ] Review coding standards and PR workflow
- [ ] Set up local development environment
- [ ] Pick up first "good first issue" with mentor support

#### Product Managers
- [ ] Review product vision, roadmap, and OKRs
- [ ] Meet with key stakeholders
- [ ] Review customer feedback and usage metrics
- [ ] Shadow customer interviews or support calls

#### Project Managers
- [ ] Review active projects and timelines
- [ ] Access project boards and tracking tools
- [ ] Review [Risks and Communication](octoacme-risks-and-communication.md) processes
- [ ] Shadow weekly stakeholder updates

#### Scrum Master
- [ ] Observe sprint ceremonies (standup, planning, retro)
- [ ] Review team velocity and past retrospective notes
- [ ] Meet with current Scrum Master (if transitioning)
- [ ] Review Agile processes and team agreements

#### Technical Writer
- [ ] Review existing documentation and style guides
- [ ] Identify documentation gaps or outdated content
- [ ] Set up documentation tools and publishing workflow
- [ ] Shadow developer to understand feature implementation

#### UX/UI Designer
- [ ] Review design system and component libraries
- [ ] Access design tools (Figma, Sketch, etc.)
- [ ] Review past user research findings
- [ ] Meet with Product Manager to understand user personas

#### Business Analyst
- [ ] Review business requirements documentation
- [ ] Meet with key business stakeholders
- [ ] Understand data sources and reporting tools
- [ ] Review process flows and current pain points

#### Release Manager
- [ ] Review [Release and Deployment Guide](octoacme-release-and-deployment.md)
- [ ] Access release tools and CI/CD pipelines
- [ ] Review release schedule and past release notes
- [ ] Shadow a production deployment

---

## Week 2-4: Active Contribution

### For All Roles
- [ ] Begin contributing to team ceremonies
- [ ] Complete first deliverable with team support
- [ ] Build relationships across functional areas
- [ ] Provide feedback on onboarding experience

### Role-Specific Contributions

#### Developers
- [ ] Complete and merge first PR
- [ ] Participate in code reviews
- [ ] Begin taking on standard-sized issues

#### Product Managers
- [ ] Write first feature specification
- [ ] Present at product team meeting
- [ ] Begin customer outreach activities

#### Project Managers
- [ ] Lead or co-facilitate a team meeting
- [ ] Update project status in tracking system
- [ ] Draft communication to stakeholders

#### Scrum Master
- [ ] Facilitate first sprint ceremony
- [ ] Conduct one-on-ones with team members
- [ ] Address first blocker or impediment

#### Technical Writer
- [ ] Complete first documentation update or new page
- [ ] Review and edit another team member's docs
- [ ] Contribute to release notes

#### UX/UI Designer
- [ ] Complete first design mockup or iteration
- [ ] Present design to team for feedback
- [ ] Conduct usability test or user interview

#### Business Analyst
- [ ] Document first set of requirements
- [ ] Facilitate requirements workshop
- [ ] Validate completed feature against business needs

#### Release Manager
- [ ] Participate in release planning
- [ ] Co-manage a release deployment
- [ ] Document and improve release process

---

## Month 2-3: Independence & Ownership

### For All Roles
- [ ] Work independently with minimal oversight
- [ ] Mentor newer team members
- [ ] Contribute to process improvement initiatives
- [ ] Share feedback in retrospectives

---

## Continuous Growth

### Ongoing Development
- Review and update knowledge of:
  - [ ] [Project Initiation](octoacme-project-initiation.md) processes
  - [ ] [Project Planning](octoacme-project-planning.md) best practices
  - [ ] [Execution and Tracking](octoacme-execution-and-tracking.md) workflows
  - [ ] [Retrospective and Continuous Improvement](octoacme-retrospective-and-continuous-improvement.md) practices

### Cross-Functional Understanding
- [ ] Shadow team members in other roles
- [ ] Participate in cross-functional workshops
- [ ] Attend optional learning sessions
- [ ] Contribute to knowledge sharing (blog posts, tech talks)

---

## Offboarding (When Leaving Team)

- [ ] Document ongoing work and handoff tasks
- [ ] Transfer ownership of key responsibilities
- [ ] Update documentation with lessons learned
- [ ] Participate in exit interview
- [ ] Return access credentials and equipment

---

## Feedback & Improvement

This onboarding checklist should be reviewed and updated quarterly based on:
- New hire feedback
- Team retrospective insights
- Changes to roles and processes

Suggested improvements should be documented as issues or discussed in team meetings.

**Note**: For detailed role definitions and responsibilities, always refer to [OctoAcme Roles and Personas](octoacme-roles-and-personas.md).
